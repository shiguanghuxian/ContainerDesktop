ContainerDesktop 0.3.0

中文
本次更新重点增强 Docker 兼容工作流、资源详情导航和发布体验。新增独立的 Docker 兼容终端应用，并支持从主应用、菜单和文件服务入口打开；终端支持多标签、样式偏好、上下文菜单、本地化文案和高频输出合并，容器与 Machine 执行入口也可切换到兼容终端。Docker 命令转换器扩展了更多常用命令识别与转换能力，减少从 Docker CLI 迁移到 container 命令时的手工处理。

资源管理方面，镜像页面新增仓库筛选模型和更完整的导航测试，网络资源新增详情页、概览、元数据和 inspect 视图，便于查看网络配置与排障。System 页面将组件版本默认改为一行缩略信息，可按需展开为详细卡片，保持系统页更紧凑。发布流程同步增强，打包脚本会把 Docker 兼容终端作为内嵌 app 一起打包、签名并写入 appcast，更新检查也可展示更完整的发布说明。

English
This release focuses on Docker-compatible workflows, richer resource navigation, and a stronger release pipeline. ContainerDesktop now includes a standalone Docker Compatibility Terminal that can be opened from the main app, app menus, and macOS file services. The terminal supports tabs, style preferences, localized strings, context menus, and coalesced high-frequency output, and container or Machine exec flows can route through the compatibility terminal. The Docker command converter also recognizes and converts more common command shapes, reducing manual work when moving from Docker CLI commands to container commands.

Resource management is expanded with repository filtering on the Images page and new Network detail screens for overview, metadata, and inspect data. The System page now keeps component versions compact by default and lets users expand the existing detailed cards only when needed. The release workflow is also improved: packaging now embeds and signs the Docker Compatibility Terminal app, includes it in release artifacts, writes appcast metadata, and surfaces fuller release notes in update checks.
